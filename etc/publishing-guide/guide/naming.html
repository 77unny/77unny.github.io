Naming Guide

공통 규칙
  • 첫 시작에 숫자,특수문자, 대문자 사용은 지양한다. "naming" (O) "Naming" (X) "0Naming" (X) "@Naming" (X)
  • 네이밍 조합은 "형태-의미-상태-(순서)" 순서로 사용한다. img file : "btn_cancle_on_01.gif" / class="btn-cancle"
  • 언더스코어(_) : 폴더,파일,이미지에 사용
    하이픈(-) : CSS 네이밍에 사용 img file : "btn_cancle_on_01.gif" / class="btn-cancle"
  • 아이디(id) 와 클래스는(class) 따옴표(아스트로피) 사용은 지양한다. id="naming" (O) class="naming" (O) id='naming' (X) class='naming' (X)
  • 폴더,파일 일 경우 한자리 정수보다 01,02 사용 권장 "btn_cancle_on_01.gif" "btn_cancle_on_02.gif" "btn_cancle_on_1.gif" "btn_cancle_on_2.gif"
  • 클래스 일 경우 다중클래스 사용 권장 (단,3개이상 조합하지않는다.) "btn btn-cancle" "ico ico-more"
예약어 - 공통
  • 전체 묶음 
    #wrap, #wrapper
  • 상단 영역 
    #header, #head
  • 본문 영역 
    #container
  • 하단영역 
    #footer, #foot
  • 네비게이션 
    #gnb
  • 기타 네비게이션 
    #tnb(t-nav), #snb(s-nav), #fnb(f-nav)
  • 영역 레이아웃 
    .inner
기본 클래스 예약어

다중클래스 사용 권장 (class="btn btn-@" 조합하여 공용으로 관리 단, 최대 3개이상 사용하지 않는다.)

  • section 태그 클래스 예약어 
    .section-@
  • article 태그 클래스 예약어 
    .article-@  .article-head .article-body
  • 네비게이션 클래스 예약어 
    .nav-@
  • 탭 클래스 예약어 
    .tab-@
  • 리스트 클래스 예약어 
    .list-@
기타 예약어

다중클래스 사용 권장 (class="btn btn-@" 조합하여 공용으로 관리 단, 최대 3개이상 사용하지 않는다.)

  • 버튼 예약어

  • 아이콘 예약어
